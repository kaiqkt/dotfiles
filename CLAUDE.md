## Overview

macOS development environment dotfiles for a Kotlin/Java/Go developer. Uses dotbot for symlink management, AeroSpace as window manager, Sketchybar as status bar, and flavours for Base16 theme management.


## Theme System (flavours)

Themes are Base16 palettes applied via `flavours apply <scheme>`.

Targets configured in `flavours/config.toml`:
- `~/.config/kitty/colors.conf`
- `~/.config/sketchybar/colors.sh` â†’ hook: `sketchybar --reload`
- `~/.config/tmux/config/theme.conf` â†’ hook: `tmux source-file ~/.config/tmux/config/theme.conf`
- `~/.config/borders/bordersrc` â†’ hook: `~/.config/borders/bordersrc` (borders reloads on re-invocation)
- `~/.fzfrc`

Generate palette from wallpaper:
```bash
python3 scripts/generate-wallpaper-theme wallpaper/wallpaper.jpg
flavours apply wallpaper
```

## Sketchybar Conventions

**Colors** â€” all defined in `sketchybar/colors.sh`, generated by flavours via `flavours/templates/sketchybar/templates/default.mustache`. Never hardcode hex values in plugins or sketchybarrc â€” use exported variables.

Key color variables and their Base16 slots:

| Variable | Slot | Semantic use |
|----------|------|--------------|
| `$BAR_BORDER_COLOR` | `0xCC` + `base03` | Bar border + graph/slider accent (synced with inactive window border) |
| `$ITEM_BG_COLOR` | `0x4a` + `base01` | Item backgrounds |
| `$RED` | `base08` | High CPU, low battery |
| `$ORANGE` | `base09` | Medium CPU, medium battery |
| `$YELLOW` | `base0A` | Medium-low CPU |
| `$GREEN` | `base0B` | Music label |
| `$BLUE` | `base0D` | Active borders (borders app) |
| `$WHITE` | `0xffffffff` | Labels |

**Fonts** â€” all defined in `sketchybar/fonts.sh` as exported variables. Never hardcode font strings in sketchybarrc or plugins.

| Variable | Font |
|----------|------|
| `$ICON_FONT` | SF Pro Display:Regular:14.0 |
| `$LABEL_FONT` | SF Pro Display:Semibold:13.0 |
| `$SMALL_FONT` | SF Pro Display:Semibold:9.0 |
| `$KNOB_FONT` | SF Pro Display:Regular:8.0 |
| `$APP_ICON_FONT` | sketchybar-app-font:Regular:14.0 |

**Plugin scripts:**
- Shebang: always `#!/bin/bash`
- Source colors at top: `source "$CONFIG_DIR/colors.sh"`
- `$NAME`, `$CONFIG_DIR`, `$SENDER` are injected by sketchybar at runtime
- No hardcoded paths

## Git Worktree Helpers (`wt:*`)

Requires a bare repo setup: `ProjectName/.bare/` + worktrees as sibling dirs.

```bash
wt:init <remote-url> [folder]   # Clone as bare repo
wt:add <branch>                 # Checkout existing remote branch
wt:create <branch> [base]       # New branch + worktree
wt:list                         # List worktree branches
wt:info                         # Detailed table (git, docker, ports, log)
wt:update                       # Fetch all from origin into .bare
wt:rename <old> <new>           # Rename dir + branch
wt:remove <branch> [--force]    # Stop containers, remove, prune, delete branch
wt:cleanup                      # Interactive per-worktree delete prompt
wt:config [key [value]]         # Read/write wt.* config on bare repo
wt:help                         # Full help
```

Each worktree gets unique `APP_PORT / DB_PORT / REDIS_PORT` in `.env`. Optional `.wt-postcreate` hook runs after creation.

## Adding a New Tool

1. Add package to `homebrew/Brewfile`
2. Create config directory (e.g. `newtool/`)
3. Add symlink to `install.conf.yaml`
4. Add the **same symlink** to the `SYMLINKS` array in `uninstall`
5. Add alias to `zsh/config/aliases.sh` if needed

## macOS Defaults Applied by install

- Fast key repeat (KeyRepeat=2, InitialKeyRepeat=15)
- Dock auto-hide with no delay
- Disable space auto-rearranging
- Finder: show hidden files, path bar, status bar, all extensions, no icon on desktop
- Disable auto-correct, window animations, extension change warning
- Hide menu bar

## Commit Conventions

Follows [Conventional Commits v1.0.0](https://www.conventionalcommits.org/).

Format: `<type>[optional scope]: <description>`

| Type | Use when |
|------|----------|
| `feat` | New feature |
| `fix` | Bug fix |
| `docs` | Documentation only |
| `style` | Formatting â€” no logic change |
| `refactor` | Restructure without feature or fix |
| `perf` | Performance improvement |
| `chore` | Tooling, dependencies, config |
| `ci` | CI/CD changes |

Breaking changes: append `!` before colon and add `BREAKING CHANGE:` footer.
